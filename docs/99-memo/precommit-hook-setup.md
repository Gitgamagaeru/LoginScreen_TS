# pre-commitフレームワーク設定タスクの洗い出し (提案)

本プロジェクトでは、Huskyよりも汎用性が高く、強力なプラグインエコシステムを持つ **pre-commit (Python製フレームワーク)** を採用し、コード品質向上とドキュメント管理の自動化を実現します。

## 1. 採用ツールとメリット

| ツール | 役割 | 採用理由 |
| :--- | :--- | :--- |
| **pre-commit** | Gitフックの管理基盤 | Python製で汎用性が高く、多言語混在にも強い。環境の隔離が自動。 |
| **pre-commit-hooks** | 基本的なファイルチェック | 余計な空白、ファイル末尾、YAML構文、巨大ファイルなどを標準プラグインでカバー。 |
| **Prettier / ESLint** | コード品質・整形 | JS/TSプロジェクトの標準ツールを `local` フックとして統合。 |
| **commitlint** | コミットメッセージの形式確認 | Conventional Commits の遵守を強制。 |

## 2. 具体的な設定タスク一覧

### フェーズ1: ツール導入と基本設定
- [ ] **pre-commitのインストール**: `pip install pre-commit`
- [ ] **設定ファイルの作成**: `.pre-commit-config.yaml` をプロジェクトルートに作成。
- [ ] **Gitフックの有効化**: `pre-commit install` の実行。

### フェーズ2: 標準フックとコード品質の自動化
- [ ] **標準フックの設定**: `trailing-whitespace`, `end-of-file-fixer`, `check-yaml`, `check-added-large-files` の導入。
- [ ] **JS/TS系チェックの統合**: コミット時に `npm run lint` と `prettier` を実行するように設定。

### フェーズ3: ドキュメント管理の自動化 (TODO.mdの要件)
- [ ] **updatedフィールドの自動更新スクリプト作成**: 
  - ステージングされた `.md` ファイルの `updated: YYYY-MM-DD` を今日の日付で置換するスクリプトを `scripts/` に作成。
- [ ] **commit-msgフックの設定**: コミットメッセージの形式チェックを `pre-commit` から呼び出す設定。

## 3. 導入後の開発フローの変化

これまでの手動チェック項目が、Gitコミット時に自動的に検証・修正されるようになります。

1. **多角的な自動ガード**: コードの文法だけでなく、ファイル末尾の空行や巨大ファイルの混入といった「リポジトリ全体の清潔さ」が保たれます。
2. **ドキュメントの鮮度維持**: `updated` 日付を意識する必要がなくなり、常に実装と連動した日付が記録されます。
3. **チーム間での環境差解消**: ツール類が `pre-commit` によって分離された環境で実行されるため、開発者ごとの設定差によるトラブルが減ります。
