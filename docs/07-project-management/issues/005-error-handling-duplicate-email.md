## 🎯 目的・背景

同一メールアドレスでの重複登録（Googleとパスワードの混在等）を防ぎ、ユーザーに分かりやすいメッセージを表示してUXを向上させるため。(ADR-002 準拠)

## ✅ やること（Todo）

- [ ] バックエンドでのメールアドレス重複チェックロジックの実装
- [ ] NextAuth のカスタムエラーハンドリング設定
- [ ] UI上に「既に〇〇で登録されています」等のカスタムメッセージを表示

## 🏆 完了条件（Acceptance Criteria）

- [ ] 既に登録済みのメールアドレスで別方式の登録を試みた際、適切にブロックされること
- [ ] ユーザーが次に何をすべきか（例: Googleでログインし直す）が伝わるエラーメッセージが表示されること
