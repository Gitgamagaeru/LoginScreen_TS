## ADR-004: ログイン・登録成功後のリダイレクト先

### ステータス

承認済み

### コンテキスト

ログインまたは新規登録が成功した後に、ユーザーをどの画面（URL）に遷移させるかが決定されておらず、認証完了後にユーザーが迷子になる懸念がありました。

### 決定

ログイン・登録成功後は、保護されたトップページである `/dashboard` に遷移させる挙動（案A）を採用しました。

### バージョン選定理由

- **なぜこのバージョンか**: 一般的なSaaSやWebアプリケーションの標準的なフローであり、ユーザーに「ログインしたこと」を明確に伝えるためです。
- **AIカットオフ対策**: なし
- **非推奨API回避**: なし
- **LTS/サポート期間**: なし

### 代替技術との比較

| 技術/手法 | 採用/却下 | 理由 |
| :--- | :--- | :--- |
| ランディングページ (`/`) に戻す（案B） | 却下 | ログイン前後での画面の区別がつきにくく、UXの低下につながるため。 |

### 影響

- **ポジティブ**: ユーザーが認証に成功してサービス内部にアクセスしたことが視覚的にも機能的にも明確になり、UXが向上します。
- **ネガティブ**: MVPとして最低限 `/dashboard` にあたる画面を作成し、認証済みユーザーからのアクセスであることをチェックするミドルウェア等の実装が必要です。

### AIへの指示

ログイン時の処理および新規アカウント作成時の処理では、認証成功時のコールバックやリダイレクト先を `/dashboard` に設定し、該当ページはセッション情報が必要となるようにNext.jsのMiddlewareなどで保護してください。
