## ADR-005: MVPで実装するソーシャルログインプロバイダーの対象

### ステータス

承認済み

### コンテキスト

ソーシャルプロバイダーの具体的な実装対象が確定しておらず、必要以上のプロバイダーを実装することによって開発スコープが広がり、工数や各種APIキー取得・設定の手間が増加する懸念がありました。

### 決定

MVPでは開発工数を最小限に抑えるため、利用者の最も多い Googleログイン のみを実装すること（案A）を決定しました。

### バージョン選定理由

- **なぜこのバージョンか**: MVPとしてのスピード感を最重視するため。また、Googleプロバイダーの連携はNextAuthにおいて標準的かつシンプルであるため。
- **AIカットオフ対策**: 特になし。Google Cloud ConsoleでのOAuth同意やクライアント情報取得の一般的な手順に沿います。
- **非推奨API回避**: なし
- **LTS/サポート期間**: なし

### 代替技術との比較

| 技術/手法 | 採用/却下 | 理由 |
| :--- | :--- | :--- |
| GoogleとGitHubの両方を実装（案B） | 却下 | GitHub等、追加のプロバイダー設定工数やテストケースが増大するため、初期フェーズでは見送りました。 |

### 影響

- **ポジティブ**: 実装や検証のコンテキストを削減し、MVPの提供スピードを高めることができます。
- **ネガティブ**: Googleアカウントを持たないユーザーに対しては、パスワード認証以外の選択肢を提供できません（次フェーズ以降でプロバイダ追加を検討）。

### AIへの指示

NextAuthにおけるプロバイダー配列には `GoogleProvider` のみを設定し、環境変数 `GOOGLE_CLIENT_ID` や `GOOGLE_CLIENT_SECRET` に基づき認証を行うように構成してください。ソーシャルログインの追加のUI要素もGoogleのみで構成します。
