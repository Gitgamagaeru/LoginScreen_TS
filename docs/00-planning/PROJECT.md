# PROJECT.md

## ビジョン
[ユーザーのログイン機能を実装する]

## ターゲットユーザー
- ペルソナ1：[システムの知識のないユーザー]
- ペルソナ2：[システムの知識のあるユーザー]

## 主要機能
### MVP（必須）
- [ ] メールアドレスとパスワードによる新規登録・ログイン
- [ ] メール送信機能（登録時の確認メール等）
- [ ] ソーシャルログイン（Google連携など、NextAuthで容易に実装可能なもの）
- [ ] ログアウト機能・ログイン状態によるアクセス制御（保護されたページ）

### Phase 2
- [ ] パスワードリセット機能
- [ ] ユーザー情報の編集（プロフィール画像や名前の変更）

## 非機能要件
- セキュリティ：パスワードの強力なハッシュ化保存（自動対応）。不正ログイン対策（トークンの安全な管理、入力値の検証）。エラーメッセージによる情報漏洩の防止。
- パフォーマンス：ログイン、画面遷移のレスポンスは2秒以内を目標とする。二重送信防止（ローディング表示）。
- 可用性：Vercelなどの安定したホスティングとマネージドデータベースを活用し、ダウンタイムを最小限にする。

## スコープ外
- 今回明示的に作らないもの：
  - 管理者用の複雑なユーザー・権限管理画面
  - 有料プランや決済機能
  - SMS等を用いた複雑な二段階認証（2FA）
